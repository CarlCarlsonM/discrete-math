Descomposición de números en sumas de cuadrados

Autor: Carlos Arturo Murcia Andrade

Objetivo

Este Jupyter Notebook busca mostrar que un número puede representarse como una suma de cuadrados (la menor cantidad de cuadrados posible).

Definiciones

Suma de cuadrados

Un número puede ser representado como una suma de cuadrados (de otros números menores que el mismo, estos números elevados al cuadrado). He aquí algunos ejemplos:

  • N = 13 = 3^2 + 2^2, entonces la menor cantidad de cuadrados que pueden sumarse para dar 13 es 2.
  • N = 100 = 10^2, por lo que la menor cantidad de cuadrados que pueden sumarse para dar 100 es 1.

Nota: Teniendo en cuenta el Teorema de los 4 cuadrados de Lagrange, el máximo “número mínimo” de cuadrados cuya suma sea N es 4. Es decir, la mínima cantidad de cuadrados para sumar un número puede ser entre 1 y 4.

Teorema de los 3 cuadrados de Legendre

Un número natural puede ser representado como la suma de 3 cuadrados: N = a^2 + b^2 + c^2. Esto siempre que n no pueda ser expresado de la forma (4^a) * (8b + 7) para números a y b no negativos.

Algoritmo 1: parecía que sí… pero no

Un acercamiento que se planteó en un inicio fue el de verificar de manera cíclica (con un greedy algorithm) la cantidad de cuadrados que sumados diesen el número de entrada [N]. En cada iteración se revisa la raíz cuadrada del número [sqrt(N)] y toma el resultado para agregarlo en un arreglo. Acto seguido, se resta el resultado de la raíz elevada al cuadrado [sqrt(N)^2] al número de entrada [N]. Luego, se sigue haciendo los mismos cálculos (los dos mencionados anteriormente) en la siguiente (y consecutivas) iteración(es) hasta que el número restante [N] sea 1.

Este método podría funcionar para ciertos números (particularmente, en números pequeños). Sin embargo, se descubrió que son más los números en los que el método no funciona que los que sí (esto puede verse evidenciado en números grandes).

Dado esto, se decidió denominar la función “get_list_of_squares_for_minimum_sum_wrong” para mostrar este comportamiento. A continuación se pueden evidenciar ciertos ejemplos que muestran más de 4 cuadrados (pueden también verse en el siguiente código):

  • 119 :-> [10, 4, 1, 1, 1] :-> 5 cuadrados

    # Return the list of squares

Algoritmo 2: el definitivo

Este algoritmo es implementado por la función "minimum_amount_of_squares". El proceso de solución es el siguiente:

  • Si el número es una raíz cuadrada exacta, la cantidad de cuadrados es 1.
  • Se saca la primera raíz cuadrada, si no es exacta, se resta el número N al primer cuadrado sacado (sqrt(N)^2). Si el resultado de esa operación es una raíz cuadrada exacta, la cantidad de cuadrados es 2.
  • Si la cantidad de cuadrados no es ni 1, ni 2, entonces, se revisa si el número es de la forma (4^a)*(8b + 7). Para ello:
    • Se divide el número N cuantas veces como sea posible para remover factores de 4.
    • Una vez se hayan removido esos factores, se verifica si el residuo de ese número dividido 8 es diferente de 7:
      • Si así es, la cantidad de cuadrados es 3.
      • Si no, la cantidad de cuadrados es 4.

Se prueban varios números para probar la efectividad de este algoritmo, y sus resultados se pueden ver en el código (y su ejecución) a continuación.

import math

'''
Description: Function to determine the minimum amount of squares that would equate to n if summed.

Input: a number (an integer) -> n.

Output: an number (an integer between 1 and 4).
'''
def minimum_amount_of_squares(n):
    # If n is negative, then, there are no squares
    if (n < 0):
        return 0
    
    # Case 1: Is n a perfect square? sqrt(n)^2 = n?
    if (n % math.sqrt(n) == 0):
        return 1
    
    # Case 2: Is n a sum of two perfect squares? a^2 + b^2 = n?
    # We first find the largest perfect square less than or equal to n, and subtract it from n to get a second term.
    first_square = int(math.sqrt(n))
    n2 = n - pow(first_square, 2)
    
    # If the second term is also a perfect square, then n is a sum of two perfect squares.
    if (n2 % math.sqrt(n2) == 0):
        return 2
    
    # Case 3: Is n not of the form (4^a)*(8b + 7)? Legendre 3 squares Theorem.
    # We divide n by 4 as many times as possible, to remove any factors of 4.
    while (n % 4 == 0):
        n = math.floor(n / 4)
    
    # Then, if the remainder when dividing n by 8 is not 7, we can represent n as the sum of 3 squares.
    if (n % 8 != 7):
        return 3
    
    # Case 4: If n is not a sum of two perfect squares, and it does have the
    # form (4^a)*(8b + 7), then it can be represented as the sum of 4 squares.
    return 4

Eficiencia del algoritmo (algoritmo 2)

El algoritmo implementado en la función "minimum_amount_of_squares" tiene una complejidad de sqrt(n), esto es, porque si el número de cuadrados es 3, entonces se entrará en un ciclo while que tendrá que descomponer el número N en factores de 4 una cantidad no mayor de la raíz cuadrada de N.

Si el número de cuadrados es 1, 2, o 4 tendrá una complejidad constante (este será, el mejor de los casos, y, por lo tanto, es irrelevante al determinar la eficiencia del método implementado).
